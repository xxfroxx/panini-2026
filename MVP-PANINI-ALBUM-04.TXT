PARTE 5 — OCR Y EXTRACCIÓN DE ID ACTUALIZADA

CORRECCIÓN ANDROID 3 — PATRÓN REAL DEL ID EN EL CROMO

Nueva información importante:
En el cromo físico, el patrón del ID normal es siempre:

3 letras + espacio + número de 1 o 2 dígitos, máximo 20

Ejemplos reales en el cromo:
COL 5
COL 05
COL 20
MEX 1
UZB 12
HAI 01

En Google Sheets, el ID está guardado sin espacio y normalmente con dos dígitos:

COL05
COL20
MEX01
UZB12
HAI01

Por tanto, para OCR debemos detectar el patrón con espacio y luego normalizarlo.

Regla para cromos normales:

* Detectar: `([A-Z]{3})\s*(\d{1,2})`
* Validar que el número esté entre 1 y 20.
* Normalizar con dos dígitos:

  * COL 5 → COL05
  * COL 05 → COL05
  * COL 20 → COL20
  * MEX 1 → MEX01
  * UZB 12 → UZB12

IMPORTANTE:
No aceptar números mayores a 20 para cromos normales.
Ejemplo:
COL 21 → inválido
ABC 99 → inválido

Para FWC:

* Detectar FWC + espacio opcional + número.
* Normalizar sin cero delante:

  * FWC 9 → FWC9
  * FWC 10 → FWC10
* No convertir FWC9 a FWC09.

Para 00:

* Detectar exactamente 00.
* Mantener como 00.

Actualizar función `extractStickerIdFromText(rawText: String): String?`.

Implementación sugerida:

fun extractStickerIdFromText(rawText: String): String? {
val text = rawText.uppercase()
.replace("-", " ")
.replace("_", " ")

```
// 1. FWC primero, porque FWC también tiene 3 letras
val fwcRegex = Regex("""\bFWC\s*(\d{1,2})\b""")
val fwcMatch = fwcRegex.find(text)
if (fwcMatch != null) {
    val number = fwcMatch.groupValues[1].toIntOrNull()
    if (number != null && number in 1..19) {
        return "FWC$number"
    }
}

// 2. Código especial 00
val zeroRegex = Regex("""\b00\b""")
if (zeroRegex.containsMatchIn(text)) {
    return "00"
}

// 3. Códigos normales: 3 letras + número 1..20
val normalRegex = Regex("""\b([A-Z]{3})\s*(\d{1,2})\b""")
val normalMatch = normalRegex.find(text)
if (normalMatch != null) {
    val prefix = normalMatch.groupValues[1]
    val number = normalMatch.groupValues[2].toIntOrNull()

    if (prefix != "FWC" && number != null && number in 1..20) {
        return prefix + number.toString().padStart(2, '0')
    }
}

return null
```

}

## PARTE 6 — EVITAR DETECCIONES REPETIDAS

El OCR puede detectar el mismo ID muchas veces por segundo.

Agregar una protección simple:

* lastDetectedId
* lastDetectionTime

Si el mismo ID se detecta otra vez dentro de 2 segundos, ignorarlo.

Ejemplo conceptual:

if (detectedId == lastDetectedId && now - lastDetectionTime < 2000) {
return
}

Esto evita que la lista se llene accidentalmente.
