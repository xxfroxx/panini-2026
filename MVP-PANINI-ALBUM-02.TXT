MVP ITERACIÓN PANINI 2026 — ANDROID 1 + APPS SCRIPT WRITE

Contexto:
Tengo una web estática de seguimiento del álbum Panini World Cup 2026.

Arquitectura actual:
Google Sheets → Apps Script → JSONP → GitHub Pages

Ya existen estas páginas:

* index.html → dashboard
* repe.html → repetidas
* falta.html → faltantes
* config.js → URL central de Apps Script

Ahora quiero crear el primer MVP Android, pero por partes.

IMPORTANTE:
Esta iteración NO incluye OCR.
Esta iteración NO incluye cámara.
Esta iteración NO incluye escaneo automático.
Solo entrada manual de ID desde Android.

---

## OBJETIVO GENERAL

Crear una app Android nativa simple en Kotlin + Jetpack Compose que permita:

1. Escribir manualmente el ID de un cromo.
2. Pulsar botón “Agregar”.
3. Enviar el ID a Apps Script mediante POST.
4. Apps Script busca ese ID en el Google Sheet.
5. Si el cromo no existe en la colección, suma la cantidad de 0 a 1.
6. Si el cromo ya existe, la app muestra confirmación:
   “Este cromo ya existe. ¿Quieres sumar +1?”
7. Si el usuario confirma, la app vuelve a llamar al backend con confirmDuplicate=true.
8. Apps Script incrementa CANTIDAD en +1.
9. Apps Script limpia cache para que index.html, repe.html y falta.html se actualicen.

## PARTE 3 — BÚSQUEDA DEL CROMO

La función addSticker debe buscar el stickerId en las hojas base del álbum.

No buscar en:

* INSTRUCCIONES
* DASHBOARD
* MASTER
* FALTANTES
* INTERCAMBIO

Sí buscar en hojas del álbum, por ejemplo:

* 01_PANINI
* 02_GRUPO_A
* 03_GRUPO_B
* ...
* 08_STICKERS_COCA_COLA
* 15_FIFA_WORLD_CUP_CHAMPIONS

El ID ya debe estar limpio en el Sheet.

Ejemplos válidos:

* COL05
* MEX01
* UZB12
* FWC9
* FWC10
* 00

IMPORTANTE:
No cambiar la estructura del Google Sheet.
No cambiar IDs.
No cambiar columnas.

La columna principal para sumar es:

CANTIDAD

El backend debe:

1. Encontrar la fila donde ID == stickerId normalizado.
2. Leer CANTIDAD.
3. Si CANTIDAD está vacío o no es número, tratar como 0.
4. Si CANTIDAD === 0, poner 1.
5. Si CANTIDAD > 0 y confirmDuplicate !== true, devolver already_exists.
6. Si CANTIDAD > 0 y confirmDuplicate === true, sumar +1.

Fe de erratas:
Aunque exista columna TIENE, la app no debe modificarla.
El backend solo debe escribir en CANTIDAD.

## FE DE ERRATAS

CORRECCIÓN MVP ANDROID 1 / APPS SCRIPT WRITE

Problema detectado:
Cuando la app Android agrega o incrementa un cromo, Apps Script también escribe en la columna `TIENE`.

Esto es incorrecto.

Regla nueva:
El backend SOLO debe actualizar la columna:

CANTIDAD

No debe escribir nada en la columna:

TIENE

Motivo:
La columna `TIENE` existe todavía en el Google Sheet, pero no debe ser modificada por la app. Por ahora todo el sistema debe basarse en `CANTIDAD`.

Cambios requeridos:

1. Buscar en Apps Script cualquier código similar a:

const tieneIndex = headers.indexOf("TIENE");

o:

sheet.getRange(rowNumber, tieneIndex + 1).setValue(...)

o cualquier escritura hacia la columna `TIENE`.

2. Eliminar o comentar esa escritura.

3. La función addSticker / update debe hacer solo esto:

* Buscar el ID.
* Leer `CANTIDAD`.
* Calcular `newQuantity`.
* Escribir `newQuantity` únicamente en la columna `CANTIDAD`.
* Limpiar cache.
* Devolver respuesta JSON.

4. No tocar `TIENE`.

La lógica correcta debe quedar así, conceptualmente:

const quantityIndex = headers.indexOf("CANTIDAD");

const oldQuantity = Number(row[quantityIndex]) || 0;
const newQuantity = oldQuantity + 1;

sheet.getRange(rowNumber, quantityIndex + 1).setValue(newQuantity);

NO hacer:

sheet.getRange(rowNumber, tieneIndex + 1).setValue(1);

NO hacer:

row[tieneIndex] = 1;

NO recalcular `TIENE`.

5. Mantener las respuestas iguales:

* added
* already_exists
* incremented
* not_found
* unauthorized

6. Mantener limpieza de cache después de actualizar `CANTIDAD`.

7. No cambiar la estructura del Google Sheet.

8. No eliminar la columna `TIENE` todavía.

9. No tocar la web.

10. No tocar Android salvo que sea necesario.

Resultado esperado:
Cuando agrego un cromo desde Android, solo cambia `CANTIDAD`.

Ejemplo:
Antes:
ID = COL05
CANTIDAD = 0
TIENE = vacío o 0

Después de agregar:
ID = COL05
CANTIDAD = 1
TIENE = queda exactamente como estaba

Si vuelvo a agregar y confirmo +1:
CANTIDAD = 2
TIENE = queda exactamente como estaba
